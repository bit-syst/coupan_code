PROJECT: Coupon Card Bulk Printing (Frontend Only - Next.js)

OBJECTIVE:
Build a Next.js frontend app where users upload an Excel file and coupon codes are printed on a fixed card design (exactly as provided). No redesign. Only dynamic value is coupon code.

TECH STACK:

* Next.js (App Router preferred)
* React Hooks
* Tailwind CSS (or plain CSS if needed)
* xlsx (for Excel parsing)

FEATURES:

1. FILE UPLOAD

* Accept only .xlsx files
* Read Excel using "xlsx" library
* Expected format:
  coupon_code
  ABC123
  XYZ456
  LMN789

2. DATA HANDLING

* Parse Excel and extract coupon_code column
* Store in state (array of strings)

3. CARD RENDERING

* Use EXACT card UI from provided design (no changes)
* Maintain:

  * Same width & height
  * Same spacing
  * Same fonts (or closest match)
* Only dynamic field:
  → Coupon code inside the empty box

4. IMPORTANT UI RULE

* Render ONE CARD PER ROW (vertical stack)
* DO NOT use grid
* DO NOT place multiple cards side-by-side
* Each card should look like an independent printable unit

5. PRINT FUNCTIONALITY

* Add "Print" button
* Use window.print()

CSS RULES (VERY IMPORTANT):

@media print {
.card {
page-break-after: always;
break-after: page;
}

body {
margin: 0;
}
}

* Each card must print on a separate page/block
* No scaling
* No distortion
* No overlapping

6. LAYOUT STRUCTURE

<App>
  <UploadSection />
  <PreviewSection>
    <Card coupon="ABC123" />
    <Card coupon="XYZ456" />
  </PreviewSection>
  <PrintButton />
</App>

7. CARD COMPONENT

<Card>
  - Fixed width/height (match design exactly)
  - Background color & layout same as image
  - Static text:
      "A Gift for you"
      "EXCLUSIVE COUPON CODE"
      QR section (static image)
  - Dynamic:
      coupon code inside box
</Card>

8. UX REQUIREMENTS

* Show preview after upload
* Scrollable list of cards
* Clean alignment
* No UI shift on print

CONSTRAINTS:

* DO NOT redesign card
* DO NOT auto-fit multiple cards per page
* DO NOT change aspect ratio
* DO NOT add responsiveness that breaks print layout

EXPECTED OUTPUT:

* Upload Excel → Cards generated instantly
* Each card shows unique coupon code
* Print → One card per page, perfectly aligned

OPTIONAL (if time):

* Add loading state during parsing
* Add error handling (invalid file)

END.
